Java Interfaces

Last modified: 5/7/23

Summary

This section covers Interfaces=Java with examples. Reference: Oracle java tutorial

  • Interfaces are akin to contracts that inform programmers how a program will act/interact. That is, it defines the standard, and others may implement how they wish.
  • Interfaces cannot be instantiated
  • Interfaces may extend other interfaces
  • May only contain constants, method declarations, default methods, static/nested types.
  • Interfaces are implemented using the "implements" keyword.
  • All methods public by default
  • All constants are: public, static, and final by default.

Using Interfaces

Defining an interface

You define an interface using the "interface" keyword
public interface InterfaceExample { .... }
They may also extend other interfaces and have constants and abstract methods:
public interface InterfaceExample extends Interface1, Interface2, Interface.....InterfaceN{

int CONSTANT_VAR = 234; //public, static and final

void doWork();
}

Using an interface

Use implements keyword as follows: public class -nameOfClass- implements -nameOfInterface-
public class Example implements Interface1 {
...
}
You must also implement all abstract methods
public interface ExampleInterface {

void doWork();
String doMoreWork();
}
-------------------------------------
public class ExampleUsage implements ExampleInterface{
//other class code

//@method to implement
public void doWork(){
//add body
}

//@method to implement
public void doMoreWork(){
//add body
return "something";
}
}

Adding content to Interfaces

Since adding methods can break old classes that implement an interface (since they need to initialize all methods), special care is taken. you can do the following:
  1. Anticipate changes in advanced
  2. Define methods as default methods
    3. //method to add to interface
    default void doSomething(){
    // you must define the method body
    }
  3. create another interface that extends the old one
    4. public interface newUpgradedInterface extends oldDeprecatedInterface {//all old and new methods}
New static methods can be added without breaking users who used the interface before change.

Interface as types

An interface is a user defined reference type When an object is instantiated from a class that implements an interface, it may use both the interface and base/superclass types.
