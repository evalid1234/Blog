Classes

Classes are the blueprints that represent real world objects. Any noun you can think of can be represented as a class. Classes are integral since they allow for the rapid and consistent construction of "instances", which are the objects themselves. Classes can have properties/fields, these are things like names, height, color etc... basically inherent traits of an object or thing. They also have methods, these are the actions the object can do, like "bark" if the object is a dog, or "fly", if its a bird. Classes are powerful since one class can quickly produce instances of objects all in one class that is easily altered/extended (if loosely coupled).

public class Dog {
//properties
private String breed;
//methods
public void bark(){ // woof } }

Classes are constructed via a constructor which in Java is a method that is inherently provided if not defined. They have the same name as the class and are public since they need to be called by other classes

public Dog(String breed){ }

Notice they have parameter options, which means users can pass arguments to the constructor for initialization

public Dog(String breed){ this.breed = breed; }

Objects

Objects are the instances of a class. If a class is a blueprint of a thing then an object is the thing. For example, in the dog class above, an object would could be a chihuahua named peter.

objects are constructed using the class name and the "new" keyword

Dog peter = new Dog("chihuahua"); Notice the '()', this calls the constructor of the class

Just like that we have a dog variable named peter that has a breed of chihuahua. Now this is a pretty poorly built class, but the idea is there, classes are useful!

Encapsulation

Encapsulation is the art of grouping related data and methods into a class. Some like to refer to it as data hiding, since often you use access modifiers like "public", "private" etc.. to limit the scope of use. This is often paired with setters and getters to fine tune the access control of the data.

public Person {
private String name;
private int age;
public int getAge();
public String getName();
}

notice how, using these modifiers you can now only get the name and age property values via public methods and you cannot alter the data yourself outside the class!

In a sense, a class dedicated to a single object is in itself an example of encapsulation since you are grouping related properties and actions to define a real life object.

Abstraction

Abstraction is the hiding of minute details. Often, programmers don't need to know how something works, they care about the big picture. One example could be a car or bus. We do not care about every component of the car like the bolts used or wiring manufactures, we want the car and it's important function. Well then, what are important functions? In the case of the car, this could be: driving, honking, braking etc.. they are broadly defined actions that are made up of many individual and hidden components. A honk requires the use of many electronics, the registering of human input and much more, but in reality we do not care how this is handled in everyday use, we just want it to make the noise. This is commonly done by limiting the scope of internal code via the "private" keyword, and using public methods to retrieve the necessary code only.

public class Car {
private void flipHonkMultiplexer(){//}
//other private methods

public void honk(){
flipHonkMultiplexer();
//other private honk internal code
}
}

In the above, the user gets the honk noise via a single public honk() call, but they never interact with the private methods that, together, produce the honk noise. Just how we like it.

Polymorphism

Polymorphism is the principle of having "many forms". There are many forms of polymorphism, overloading, overriding, type casting etc.. this is typically done in compile or run-time.

overloading is done in compile-time, this is often used in methods, but can also extend to operators.

public foo();
public foo(int bar);
public foo(String bar);

Notice we have 3 public methods all with the same name, this is possible since the compiler differentiates based on the parameter list. This allows the foo method to define different behavior based on the arguments passed, essentially taking many forms.

Run time polymorphism can also happen in many forms, it can be as simple as type casting, to passing generics and method overriding a superclass method

Casting

Dog puppy = new Shitzu();

Casting

double number = (double) 12; This would happen automatically but still a casting.

Overriding

class Parent {
void Print()
}
// Class 2
class subclass1 extends Parent {
void Print(){ //print 1 }
}
class subclass2 extends Parent {
void Print() { // Print 2 } }

If we called the functions Print from both subclasses the print would be "1" and "2" respectively. Showing how overriding occurs.

Inheritance

Classes can inherit properties and methods of another class, known as the "base" or "super" class. This allows for swift code extension and reuse, without the need for more typing. Classes and interfaces inherit using the "extends" keyword.

public class Dog{//properties and methods}
public class Chihuahua extends Dog{
//all dog code
public customChihuhuaFeature();
}
