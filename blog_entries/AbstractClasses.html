Abstract classes and methods.

Last modified: 5/7/23

Summary

This section covers abstract classes/methods in Java with examples. Reference: Oracle/tutorials/abstract classes

  • Abstract Classes are defined using "abstract" key word. Methods are declared without implementation.
  • Abstract Classes cannot be instantiated and may include abstract methods . They may also be subclasses
  • Abstract classes can include defined methods and fields.
  • Abstract classes can include defined methods and fields.
  • Abstract classes may have, static, final, public, protected, private and concrete methods.
  • Abstract classes, like other classes can only be extended once.

Using abstact classes and methods

Using abstract classes.

  • Abstract classes are defined using abstract keyword
  • Abstract classes cannot be instantiated. No "new" keyword.
  • Abstract classes can be subclassed can be extended.
  • If subclasses and the subclass does NOT implement all abstract methods, it too becomes abstract
  • If a class includes an abstract method, the class must be declared abstract No "new" keyword.
  • Abstract classes can include defined methods and fields.
